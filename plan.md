# Kitchen Brain вЂ” Application Plan

## Overview
An LLM-interfaced kitchen management app that understands natural language to handle meal planning, nutritional tracking, pantry management, and grocery shopping.

## Tech Stack
- **Language:** Python 3.11+
- **Database:** SQLite (via `sqlite3` stdlib or `sqlalchemy`)
- **LLM Integration:** `anthropic` or `openai` SDK via function-calling
- **CLI Framework:** `click` or `argparse`

## Data Model (SQLite)

```
users(id, name, calorie_goal, protein_goal, fiber_goal, vitamins_goal)
preferences(user_id, kind[like/dislike], item)
ingredients(id, name, category, default_unit)
nutrition(ingredient_id, per_unit, calories, protein_g, fiber_g, vitamins_json)

pantry(user_id, ingredient_id, quantity, unit, purchased_date, expiry_date)

recipes(id, name, cuisine, prep_time, cook_time, servings, instructions_text)
recipe_ingredients(recipe_id, ingredient_id, quantity, unit, optional)
recipe_nutrition(recipe_id, per_serving_calories, protein_g, fiber_g, vitamins_json)

meal_logs(id, user_id, timestamp, notes)
meal_log_items(log_id, ingredient_id, quantity, unit)

meal_plans(id, user_id, week_label, created_at)
meal_plan_days(plan_id, day_name, meal_type, recipe_id)

grocery_purchases(id, user_id, timestamp, source[manual])
grocery_purchase_items(purchase_id, ingredient_id, quantity, unit, price)
```

## Built-in Recipe DB
- Seed with ~50-100 common recipes across cuisines (asian, italian, american, mexican, etc.)
- Each recipe has structured ingredients + approximate nutrition
- Stored as a SQLite seed file or JSON import
- New recipes can be imported from text or generated by the LLM at any time

## Feature Modules

### 1. Recipe Suggestor (`suggest_dinner`)
Filter recipes by preferences в†’ score by pantry match в†’ compute remaining nutritional need в†’ rank suggestions.

### 2. Pantry / Grocery Tracker
- `kb add-grocery` вЂ” interactive prompt or CLI args for item + quantity
- `kb record-purchase` вЂ” batch add multiple items
- `kb pantry` вЂ” list current stock
- `kb expire` вЂ” show items nearing expiry
- Manual entry only (no OCR)

### 3. Meal Logger
- `kb log-meal "4 slices of cheese pizza"` вЂ” LLM parses natural language
- `kb day` / `kb week` вЂ” nutritional summary vs goals
- `kb history` вЂ” browse past meals

### 4. Menu Planner
- `kb plan-week` вЂ” generates a 7-day plan prioritizing pantry ingredients
- `kb plan-week --save` вЂ” persists the plan
- `kb grocery-list` вЂ” generates shopping list from current plan

## LLM Integration Pattern

For queries (read):
```
User input в†’ CLI constructs context (pantry, preferences, goals, recipes)
          в†’ LLM call with tools: search_recipes, get_recipe_detail, get_pantry, get_nutrition_remaining
          в†’ LLM reasons в†’ returns answer
```

For mutations (write):
```
User input в†’ LLM parses into structured data
          в†’ CLI confirms в†’ writes to DB
```

## Key Design Decisions

| Decision | Choice |
|---|---|
| Auth / multi-user | Single-user local app (no auth) |
| LLM provider | Configurable via env var, defaults to Anthropic |
| Recipe storage | Structured DB (not free-text) so queries are fast without LLM |
| Nutrition data | Per-ingredient nutrition + recipe-level computed totals |
| LLM role | Natural language parser + conversational layer, not the database |
| Statefulness | None вЂ” CLI is stateless; DB is the source of truth |

## Build Order

1. DB schema + seed data (ingredients, nutrition, built-in recipes)
2. CLI framework with `kb --help`
3. Pantry CRUD вЂ” add/remove/list items
4. Recipe Suggestor вЂ” match pantry + preferences в†’ suggestions
5. Meal Logger вЂ” log meals, track nutrition vs goals
6. LLM integration вЂ” wire up function-calling for natural language
7. Menu Planner вЂ” weekly plan + grocery list generation
8. Recipe import вЂ” add new recipes from text or LLM generation
