# Kitchen Butler — Application Plan

## Overview
An LLM-interfaced kitchen management app that tracks your pantry, logs meals, and suggests recipes. Designed to work with any **MCP-compatible LLM tool** (opencode, Claude Code, Cursor, Windsurf, etc.) — the LLM agent handles natural language parsing and KB is a pure data management layer.

## Tech Stack
- **Language:** Python 3.11+
- **Database:** SQLite (via `sqlalchemy`)
- **CLI Framework:** `click`
- **MCP Server:** `mcp` (FastMCP)
- **LLM:** Handled externally by any MCP-compatible tool (opencode, Claude Code, Cursor, etc.) via MCP (not embedded in KB)

## Data Model (SQLite)

```
users(id, name, calorie_goal, protein_goal, fiber_goal, vitamins_goal)
preferences(user_id, kind[like/dislike], item)
ingredients(id, name, category, default_unit)
nutrition(ingredient_id, per_unit, calories, protein_g, fiber_g, vitamins_json)

pantry(user_id, ingredient_id, quantity, unit, purchased_date, expiry_date)

recipes(id, name, cuisine, prep_time, cook_time, servings, instructions_text)
recipe_ingredients(recipe_id, ingredient_id, quantity, unit, optional)
recipe_nutrition(recipe_id, per_serving_calories, protein_g, fiber_g, vitamins_json)

meal_logs(id, user_id, timestamp, notes)
meal_log_items(log_id, ingredient_id, quantity, unit)

meal_plans(id, user_id, week_label, created_at)
meal_plan_days(plan_id, day_name, meal_type, recipe_id)

grocery_purchases(id, user_id, timestamp, source[manual])
grocery_purchase_items(purchase_id, ingredient_id, quantity, unit, price)
```

## Built-in Recipe DB
- Seed with ~25-50 common recipes across cuisines (asian, italian, american, mexican)
- Each recipe has structured ingredients + approximate nutrition
- Stored as JSON seed files and loaded at `kb init`
- New recipes can be imported from text or generated by the LLM at any time

## Feature Modules

### 1. Pantry Management
- `kb pantry` — list current stock
- `kb add <item> <qty>` — add item (merges duplicates by name+unit)
- `kb update <item> <qty>` — set item quantity (replaces all rows for that ingredient)
- `kb remove <item>` — delete item entirely
- `kb expire` — show items nearing expiry (not yet built)
- Manual entry only (no OCR)

### 2. Recipe Suggestor
- Score recipes by pantry match percentage
- Rank suggestions by match score
- Filter by cuisine
- Preferences/goal-aware filtering (not yet built)

### 3. Meal Logger
- `kb day` — nutritional summary vs goals
- `kb week` — weekly nutritional summary (not yet built)
- `kb history` — browse past meals (not yet built)

### 4. Menu Planner (not yet built)
- `kb plan-week` — generates a 7-day plan prioritizing pantry ingredients
- `kb plan-week --save` — persists the plan
- `kb grocery-list` — generates shopping list from current plan

### 5. MCP Server
- `kb serve` — starts the MCP server on stdio
- Exposes all KB operations as MCP tools for any MCP-compatible LLM tool

## LLM Integration Pattern (MCP)

```
User (natural language) → LLM tool (opencode / Claude Code / etc.) → MCP tools (stdio)
                                                                      ↓
                                                                Kitchen Butler MCP Server
                                                                      ↓
                                                                SQLite DB
```

KB is purely a data management layer. All LLM reasoning, natural language parsing, and conversation management is handled by the LLM tool. The user never calls KB tools directly — they talk to the LLM agent in natural language, and the agent uses KB's MCP tools to read/write data.

## Key Design Decisions

| Decision | Choice |
|---|---|
| Auth / multi-user | Single-user local app (no auth) |
| LLM provider | Handled by the MCP-compatible tool (not KB's concern) |
| LLM integration | MCP stdio transport (no network setup needed) |
| Recipe storage | Structured DB (not free-text) |
| Nutrition data | Per-ingredient nutrition + recipe-level computed totals |
| LLM role | Natural language parser only — not the database |
| Statefulness | None — CLI/MCP is stateless; DB is the source of truth |

## Build Order

1. DB schema + seed data (ingredients, nutrition, built-in recipes)
2. CLI framework with `kb --help`
3. Pantry CRUD — add/update/delete/list with merge-on-add
4. Recipe Suggestor — pantry-match scoring → ranked suggestions
5. Meal Logger — log meals, track nutrition vs daily goals
6. MCP Server — expose all operations as MCP tools
7. Tests — comprehensive test suite for all tools
8. Menu Planner — weekly plan + grocery list generation
9. Recipe import — add new recipes from text or LLM generation
